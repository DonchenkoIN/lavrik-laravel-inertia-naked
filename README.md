## Установка
- composer install
- npm install
- cp .env.example .env
- php artisan migrate --seed (если не использовать дамп БД)

## Мои пояснения
Я исходил из того, что url это по сути slug, формируемый из title поста.
user_id поста устанавливается рандомно при создании поста и сохраняется при редактировании. Этот механизм выбран, поскольку у нас нет механизма авторизации. В дальнейшем это будет реализовано в соответствии с будущими требованиями
Не реализован механизм внедрения нейминга маршрутов на клиенте, поскольку возникли сложности в установке Ziggy. Этот вопрос остаётся открытым на данном этапе.
